1. Executive Summary

Dokumen ini mendefinisikan arsitektur tingkat tinggi (High-Level Architecture) secara komprehensif untuk sistem hPanel (Hostinger/Niagahoster). Arsitektur ini tidak hanya mencakup aliran data analitik, tetapi juga memetakan infrastruktur inti, lapisan keamanan (Security), *Polyglot Persistence* lanjutan, serta sistem observabilitas (Monitoring) yang mendasari operasional jutaan pengguna di seluruh dunia secara *real-time*.

Pembaruan v3.0 (Masterpiece): Menambahkan komponen krusial setara tingkat Enterprise nyata, meliputi: DNS & Load Balancer, Centralized Logging (ELK Stack), Cold Storage Backup (Amazon S3), Sistem Penjadwalan (Cron Scheduler), serta alur CI/CD Deployment.

3. Architecture Component Definition

Domain Komponen Utama Fungsi Sistem (System Function)
Edge & Security Layer DNS, WAF, Load Balancer, API Gateway Menangani resolusi domain (Cloudflare DNS), mendistribusikan beban lalu lintas (NGINX), mitigasi serangan DDoS, otentikasi JWT, dan pembatasan laju (Rate Limiting).
Core Processing Layer Backend API, Elasticsearch Mengeksekusi logika bisnis utama hPanel. Elasticsearch terintegrasi untuk pencarian teks penuh berkecepatan tinggi (seperti pencarian aset domain dan basis pengetahuan).
Async & Scheduled Ops Cron Scheduler, RabbitMQ, Workers Menjalankan proses latar belakang. Ditambahkan Apache Airflow / Cron untuk mengeksekusi tugas terjadwal otomatis (contoh: pembuatan tagihan bulanan).
Polyglot Storage & Infra PostgreSQL, Redis, CephFS, LVE, Amazon S3 Sistem persisten hibrida. Dilengkapi Amazon S3 (Cold Storage) sebagai repositori backup jangka panjang yang terisolasi dari CephFS utama untuk mitigasi bencana.
Data Analytics Pipeline Kafka, CDC, ClickHouse, BigQuery Apache Kafka menangani aliran data analitik bervolume ekstrem. ClickHouse menelan metrik server *real-time*, dan BigQuery berfungsi sebagai DWH.
Observability & Logging ELK Stack, Prometheus, Grafana, PagerDuty Sistem pemantauan 360 derajat. ELK Stack bertugas melakukan sentralisasi log teks aplikasi, melengkapi metrik Prometheus untuk keperluan investigasi (debugging) *engineer*.
DevOps & Deployment GitLab CI/CD, Kubernetes Infrastruktur otomatisasi di belakang layar yang memungkinkan para *Developer* untuk merilis pembaruan kode aplikasi (Continuous Deployment) secara aman tanpa *downtime*.

4. Design Principles & Justification

Desain ini menganut prinsip Decoupled Architecture dan Polyglot Persistence. Kelemahan pada arsitektur sebelumnya (Single Point of Failure pada Message Broker) telah diselesaikan dengan memisahkan *Task Broker* (RabbitMQ) untuk operasional dan *Event Stream* (Kafka) untuk analitik. Penambahan blok Infrastructure (CephFS/LVE) merefleksikan realitas operasional perusahaan web hosting. Selain itu, lapisan Security dan Observability menjamin bahwa sistem tidak hanya berfungsi (*functional requirement*), tetapi juga tangguh, aman, dan mudah dipantau (*non-functional requirements*).
